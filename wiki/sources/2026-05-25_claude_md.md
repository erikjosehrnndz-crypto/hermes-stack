## Hechos clave extraidos

- Push HTTPS es el unico metodo que funciona en este VPS (SSH no configurado)
- GitHub username exacto: `erikjosehrnndz-crypto` (con sufijo `-crypto`)
- Patron obligatorio: incrustar token en URL → push → restaurar URL limpia
- Commit por tarea logica, no por sesion — riesgo de perdida por rate limit
- Commit ANTES de lanzar orquestacion multi-agente
- Leer el archivo completo antes de editar — nunca appendear sin leer el final
- Sub-agentes no heredan permisos Write/Bash del padre
- Sub-agentes no deben editar CLAUDE.md (security warning); el orquestador sintetiza
- Haiku para investigacion, Sonnet para sintesis, Opus solo si el usuario lo activa
- Next.js 14.2.x usa `next.config.mjs`, nunca `.ts` (solo Next.js 15+)
- Componentes con hooks requieren directiva `'use client'` como primera linea
- `public/.gitkeep` debe existir para que el Docker build standalone no falle
- Health check de LiteLLM requiere header `Authorization: Bearer $LITELLM_MASTER_KEY`
- LaTeX con babel espanol + TikZ: siempre incluir `babel` primero en usetikzlibrary
- YAML no existe en listings de LaTeX — definir lenguaje en preambulo
- Fuentes disponibles en VPS: DejaVu. fontawesome5 no instalado.
- Compilacion XeLaTeX: 3 pasadas + verificacion de errores con `grep "^!" main.log`
- Archivos nunca al repo: `*.db`, `*.zip`, `Sync/`, `snap/`, `*.aux/*.toc/*.out`, `node_modules/`, `.next/`, `.env`
